# Shared Catdex Migration Plan

## 목표

냥도감을 사용자별 개인 도감에서 동네 사용자들이 함께 만드는 공유 도감으로 전환한다.

핵심 원칙:

- 고양이 자체는 공유 엔티티로 관리한다.
- 사용자의 수집 여부와 발견 기록은 사용자별로 관리한다.
- 정확한 위치는 저장하지 않고 지역 단위만 저장한다.
- 모든 public 테이블은 RLS를 활성화한다.
- 구현은 단계별로 진행하고 각 단계마다 검증한다.
- 커밋은 가장 작은 논리 단위로 나누고, 기능 단위 완료 후 PR을 만든다.

## 현재 상태

- `cats`는 `user_id`를 가진 사용자별 개인 고양이 테이블이다.
- `cat_encounters`는 사용자별 발견 기록이다.
- `region_cats`도 사용자별 지역-고양이 연결이다.
- RLS가 `auth.uid() = user_id` 기준이라 다른 사용자의 고양이는 보이지 않는다.
- 결과적으로 현재 앱은 공유 도감이 아니라 개인 도감이다.

## 목표 기능

1. 공유 고양이 등록
   - 로그인 사용자는 새 고양이를 공유 도감에 등록할 수 있다.
   - 최초 등록자는 `created_by`로 기록한다.
   - 등록 시 첫 발견 기록도 함께 생성한다.

2. 기존 고양이 재발견
   - 공유 고양이 상세에서 "나도 봤어요"로 발견 기록을 추가한다.
   - 발견 기록은 `cat_encounters`에 사용자별로 저장한다.
   - 공유 고양이의 총 발견 수와 최근 발견일을 갱신한다.

3. 내 도감
   - 공유 도감 전체와 별개로 내가 발견한 고양이만 모아 보여준다.
   - `user_cat_collections`로 사용자별 수집 상태를 관리한다.

4. 지역 도감
   - 지역별로 발견된 공유 고양이 목록을 보여준다.
   - 지역 중심점과 반경만 사용하고 정확한 좌표는 저장하지 않는다.

5. 사진
   - 고양이 대표 사진과 발견 기록 사진을 분리한다.
   - 공개 사진은 공유 조회 가능하게 하되, 삭제/수정은 업로더 또는 관리자만 가능하게 한다.

6. 신고/관리
   - 중복 고양이, 부적절한 사진, 위치 노출 위험, 잘못된 정보 신고를 받을 수 있게 한다.
   - 관리자 병합/숨김/삭제는 후속 단계로 둔다.

## 데이터 모델 방향

공유 테이블:

- `cats`: 공유 고양이
- `cat_encounters`: 발견 기록
- `user_cat_collections`: 사용자별 수집 상태
- `regions`: 지역
- `cat_regions`: 공유 고양이-지역 연결
- `cat_photos`: 고양이/발견 사진
- `reports`: 신고

주요 변경:

- `cats.user_id`를 제거하고 `cats.created_by`로 변경한다.
- `cats`는 로그인 사용자가 모두 조회할 수 있게 한다.
- `cat_encounters.user_id`는 유지한다.
- 내 도감은 `user_cat_collections`에서 계산한다.
- 지역별 목록은 `cat_regions` 기준으로 계산한다.

#### Step 6 A/B 검증 seed

`supabase/seed.shared-catdex-step6.sql`은 SQL/RLS 검증용 두 사용자를 만든다. Expo 시뮬레이터에서 실제 Google/Kakao OAuth 계정으로 확인할 때는 seed 상단의 A/B UUID를 `auth.users`의 실제 OAuth 사용자 ID로 바꿔 같은 데이터 모양을 만들면 된다.

- 사용자 A: `step6-a@example.test` / `password-a`
- 사용자 B: `step6-b@example.test` / `password-b`

seed 기대 상태:

- 공유 도감 전체: `스텝감자`, `스텝나비`, `스텝후추`가 A/B 모두에게 보인다.
- A 내 도감: `스텝감자`, `스텝나비`만 보인다.
- B 내 도감: `스텝나비`, `스텝후추`만 보인다.
- `스텝나비`는 공유 발견 횟수 2회이고, A/B 둘 다 수집 상태를 가진다.
- `스텝후추`의 비공개 발견 기록은 B 본인에게만 보여야 한다.
- 지도는 `Step6 중동 근처`, `Step6 상동 근처`에서 공유 고양이 연결을 보여야 한다.
- 도감 단서 카드는 `cat_sightings`의 open 목격 제보를 미확인 카드로 보여야 한다.

#### Step 6 화면 체크리스트

홈:

- A/B 로그인 모두 공유 도감 최신 고양이 목록에 같은 shared cats가 보인다.
- `공유 도감 전체` 수치는 A/B가 같다.
- `내 도감 수집`, `이번 주 내 수집`, `내 최근 재발견`은 사용자별 `user_cat_collections` 기준으로 달라진다.

도감:

- 카드 목록의 발견된 공유 고양이는 A/B 모두 같은 `cats` 조회 결과를 기반으로 한다.
- 상단 수집 진행률은 사용자별 내 도감 개수 기준이다.
- 미확인 카드는 `cat_sightings.status = 'open'`인 목격 제보를 보여준다.
- 미확인 카드 클릭 시 정확한 좌표가 아니라 지역 단서만 노출된다.

촬영:

- 기존 고양이 재발견 목록은 공유 고양이 기준으로 나온다.
- `봤어요`를 누르면 새 공유 고양이를 만들지 않고 `record_cat_encounter`로 발견 기록과 내 수집 상태가 갱신된다.
- 새 고양이 등록은 `create_cat` RPC로 공유 고양이, 첫 발견, 내 수집, 지역 연결을 함께 만든다.
- 목격 제보 등록은 `create_cat_sighting` RPC로 미확인 단서만 만든다.

지도:

- 지역별 고양이는 사용자별 `region_cats`가 아니라 공유 `cat_regions` 기준으로 집계된다.
- A/B 모두 같은 지역별 공유 고양이 목록을 본다.
- 화면에는 지역명/반경 기반 정보만 보이고 정확한 좌표 입력 또는 저장 흐름은 없다.

MY:

- 내 도감 목록은 `user_cat_collections` 기준이므로 A와 B가 다르다.
- 프로필 발견 수와 재발견 횟수는 내 수집 상태 기준으로 계산된다.
- 공유 도감에 존재하지만 내가 수집하지 않은 고양이는 MY 내 도감에 나오지 않는다.

#### Step 6 코드 점검 결과

- 홈 요약 타입과 API를 `shared*` / `my*` 필드로 분리해 공유 도감 수치와 내 수집 수치가 섞이지 않게 했다.
- 도감 단서 카드의 API 매핑을 `regionHint` 기준으로 맞춰 미확인 목격 제보가 타입 오류 없이 표시되게 했다.
- `CatCard.tsx`의 병합 충돌 마커를 제거하고 미발견 단서 카드는 잠금 스타일을 유지하게 했다.
