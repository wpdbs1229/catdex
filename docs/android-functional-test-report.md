## 2026-05-20 후속 수정: 검색, MY 메뉴, 등록 폼

### 문제 정의

1. 도감 검색
   - 검색바가 `TextInput`이 아닌 정적 텍스트라 사용자가 검색어를 입력할 수 없었다.
   - 검색어 상태가 없어 도감 카드와 미발견 슬롯을 필터링하는 로직도 없었다.

2. MY 메뉴
   - `탐험 기록`, `내가 공유한 도감`, `좋아요한 도감` 항목에 `onPress`가 연결되어 있지 않았다.
   - 앱 내 단일 화면 상태 타입에도 해당 목적지 화면이 없어 눌렀을 때 이동할 수 있는 구조가 없었다.

3. 등록 폼
   - `도감에 등록하기`와 `미확인 제보로 남기기` 버튼이 `isSubmitting`만 보고 활성화됐다.
   - 이름과 발견 장소가 공백이어도 제출 payload가 그대로 전달될 수 있었다.

### 해결 단계

1. 도감 검색
   - 검색바를 `TextInput`으로 교체했다.
   - 검색어 상태를 추가하고 이름, 번호, 털색, 관계, 태그, 메모 기준으로 도감 카드를 필터링한다.
   - 미발견 슬롯도 털색, 지역 힌트, 행동 힌트, 단서 문구 기준으로 함께 필터링한다.
   - 검색 결과가 없을 때 별도 빈 상태 문구를 표시한다.

2. MY 메뉴
   - 네비게이션 타입에 `explorationHistory`, `sharedCollections`, `likedCollections` 화면을 추가했다.
   - `MyPageScreen`에 세 메뉴 핸들러를 추가하고 실제 `onPress`로 연결했다.
   - 탐험 기록, 내가 공유한 도감, 좋아요한 도감 화면을 추가해 현재 앱 데이터 기준 목록/빈 상태를 표시한다.

3. 등록 폼
   - 제출 전 이름, 발견 장소, 메모를 trim 처리한다.
   - 도감 등록은 이름과 발견 장소가 있어야 활성화된다.
   - 미확인 제보는 발견 장소가 있어야 활성화된다.
   - 필수 입력 안내 문구를 각 입력 필드 아래에 표시한다.

검증:
- `catdex-client`에서 `npm run typecheck` 통과.

## 확인한 기능 목록

### 1. 인증

- 카카오/구글 OAuth 로그인 버튼 제공
- Supabase OAuth 콜백으로 앱 복귀
- SecureStore 기반 세션 복원
- 로그아웃 메뉴 제공

상태:
- 로그인 완료 후 홈 진입 확인.
- 이전 `localhost:3000` redirect 문제는 Supabase Redirect URLs에 `catdex://auth/callback` 등록 후 해결된 것으로 확인.

### 2. 홈

- 오늘 발견, 내 도감 수집, 공유 도감 전체, 내 최고 재발견 요약
- 최근 발견 고양이 목록 또는 빈 상태
- `새 고양이 등록하기` CTA
- 정확한 좌표 대신 동네 단위 기록 안내

상태:
- 현재 계정 기준 데이터가 0건인 빈 상태가 정상 표시됨.
- CTA로 촬영 화면 진입 확인.

미통과/개선:
- 우측 상단 알림 아이콘은 `Pressable`이지만 `onPress`가 연결되어 있지 않음.
- `최근 발견한 고양이 > 더보기`는 텍스트만 있고 이동 동작이 없음.

### 3. 촬영 및 등록

- 카메라 권한 요청
- 카메라 미리보기
- 전/후면 전환 버튼
- 셔터 촬영
- 촬영 후 `다음`으로 기록 설정 화면 이동
- 기존 고양이 재발견
- 신규 도감 등록
- 미확인 제보 등록
- 털 색상/성격 태그/발견 장소/메모 입력

상태:
- 권한 요청 다이얼로그 표시 및 허용 확인.
- 카메라 미리보기 표시 확인.
- 셔터 후 `촬영 완료`, `다음` 버튼 표시 확인.
- `기록 설정` 화면 진입 확인.

미통과/개선:
- Android 에뮬레이터에서 촬영 결과가 검은 이미지로 저장됨. 미리보기는 정상이라 `takePictureAsync` 결과 또는 에뮬레이터 카메라 환경 이슈 확인 필요.
- 해결됨: `CatRegisterForm`에 이름/발견 장소 필수 검증과 공백 trim 처리를 추가함. 메모는 선택 입력으로 유지.
- 실제 `도감에 등록하기`/`미확인 제보로 남기기`는 DB에 테스트 데이터가 남기 때문에 이번 검증에서는 누르지 않음.

### 6. 지도

- 공유 도감 지도
- 지역별 공유 고양이 수
- 지역 선택
- 정확한 위치 비공개 안내
- Kakao Map WebView 실패 시 fallback 지도

상태:
- 지도 탭 진입은 성공.
- 지역 목록과 지역 선택 UI 표시.

미통과:
- 실제 Kakao Map WebView가 로드되지 않고 fallback 지도만 표시됨.
- 화면에 `Kakao 지도를 불러오지 못해 지역 단위 지도로 표시 중이에요.` fallback 상태가 나타남.

확인 필요:
- Kakao Developers Web platform 도메인에 `EXPO_PUBLIC_KAKAO_MAP_WEB_ORIGIN` 값인 `https://catdex.local` 등록 여부
- Kakao JavaScript 키가 현재 앱/도메인에서 유효한지
- Android WebView에서 `https://dapi.kakao.com/v2/maps/sdk.js` 접근 가능한지

## 우선 수정 권장 사항

1. Kakao Map WebView 로드 실패 원인 확인
   - 현재 지도 핵심 기능이 fallback으로만 동작함.
   - Kakao JavaScript 키, Web platform domain, Android WebView 네트워크 접근을 확인해야 함.

2. Android 촬영 결과 검은 이미지 문제 확인
   - 미리보기는 정상이나 촬영 결과가 검은 화면.
   - 실제 기기에서도 재현되는지 먼저 확인하고, 재현되면 `expo-camera` capture 옵션/렌더 타이밍을 수정해야 함.

3. 도감 검색바 실제 기능 구현
   - 해결됨: 입력 가능한 검색바와 카드/슬롯 필터링을 추가함.

4. MY 메뉴 미구현 항목 연결
   - 해결됨: `탐험 기록`, `내가 공유한 도감`, `좋아요한 도감` 전용 화면과 이동 핸들러를 추가함.

5. 등록 폼 필수값 검증 추가
   - 해결됨: 도감 등록은 이름/발견 장소, 미확인 제보는 발견 장소가 없으면 제출할 수 없도록 수정함.

6. RevenueCat 설정 추가
   - `EXPO_PUBLIC_REVENUECAT_ANDROID_API_KEY`와 실제 Offering/Product 설정이 없으면 냥꾸러미 구매 검증 불가.
