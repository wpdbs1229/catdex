# 냥도감 제품 기획안

작성일: 2026-05-30  
대상 앱: `catdex-client` Expo React Native 앱 + Supabase 기반 백엔드

## 1. 제품 한 줄 정의

`냥도감`은 사용자가 동네에서 만난 길고양이를 도감처럼 수집하고 꾸미는 경험을 제공하면서, 동의 기반의 정확한 관찰 위치는 내부에 안전하게 저장하고 공개 화면에는 흐림 처리된 지역 정보만 보여주는 길고양이 시민과학 플랫폼이다.

## 2. 제품 배경

길고양이는 매일 같은 골목, 같은 화단, 같은 주차장 근처에서 반복적으로 마주치지만, 대부분의 만남은 사진첩이나 기억 속에 흩어진다. `냥도감`은 이 만남을 "기록", "재발견", "수집", "공유"라는 가벼운 루프로 바꾼다.

현재 코드는 기존 개인 도감 구조에서 공유 도감 구조로 전환되어 있다. 고양이 엔티티는 동네 사용자들이 함께 보는 공유 대상으로 관리하고, 사용자의 수집 여부와 발견 기록은 사용자별로 분리한다.

현재 구현은 정확한 좌표를 저장하지 않고 지역명, 반경, 동네 단위 힌트만 사용한다. 앞으로의 제품 방향은 초반 비전 문서처럼 **정확한 GPS/EXIF 관찰 위치를 내부 민감 데이터로 저장하되, 일반 사용자와 공개 화면에는 격자/행정동/지역 힌트만 노출하는 구조**로 전환한다. 이 전환은 단순 화면 변경이 아니라 위치 동의, 데이터 모델, RLS, RPC, 감사 로그, 운영 권한 체계까지 함께 바꾸는 별도 단계로 다룬다.

### 2.1 초기 비전 문서에서 반영한 근거

초기 기획안의 핵심은 “고양이를 많이 찍게 만드는 앱”이 아니라, 반복 촬영된 사진과 위치·시간·관찰 태그를 현장에서 쓸 수 있는 데이터로 정제하는 것이다. 현재 문서는 이 방향을 제품 개발 기준으로 채택한다.

- TNR 현장에서는 귀끝 표식, 외형, 건강 상태, 포획/방사 위치, 반복 출현 여부가 실제 판단 자료가 된다.
- 지자체와 동물보호 단체는 이미 지역별 서식현황, 자묘 여부, 중성화 여부, 민원 현황, 주변 환경 데이터를 필요로 한다.
- 사진 기반 개체 식별은 같은 고양이가 언제 어디서 다시 관찰되는지 추적하는 시민과학 데이터가 될 수 있다.
- 공공 TNR 데이터와 앱 관찰 데이터를 연결하면 `현장 발견 -> TNR 진행 -> 방사 후 재출현 확인` 흐름을 만들 수 있다.
- 따라서 B2C 도감 UX는 데이터 수집의 입구이고, 장기 제품은 TNR·건강·서식·민원 대응 데이터를 안전하게 축적하는 플랫폼이다.

### 2.2 포지셔닝

| 겉으로 보이는 앱 | 내부 데이터 플랫폼 |
| --- | --- |
| 길고양이 사진을 찍어 도감을 모음 | 개체별 반복 관찰 기록 축적 |
| 고양이 카드, 표지, 배지, 도장 꾸미기 | TNR 여부, 건강 상태, 자묘 출현, 서식환경 데이터 수집 |
| 동네 고양이 탐방 | 동네별 미중성화/번식/구조 우선순위 지도 |
| 공개 도감과 좋아요/팔로우 | 인증 활동가·단체·지자체용 관리 대시보드 기반 |

개발 우선순위는 “B2C는 귀엽고 가볍게, 데이터 구조는 진지하게, 위치 공개는 보수적으로”라는 원칙을 따른다.

### 2.3 사용자가 냥도감을 선택해야 하는 이유

`냥도감`의 선택 이유는 "고양이 사진을 올릴 수 있다"가 아니다. 사진첩에는 순간만 남고, SNS에는 게시물만 남고, 동네 커뮤니티에는 글이 흩어진다. 냥도감은 **동네 고양이 한 마리의 기록이 계속 쌓이고, 다시 만났을 때 알아볼 수 있게 해주는 도감**이어야 한다.

핵심 가치제안:

> 냥도감은 내가 만난 길고양이를 안전하게 기록하고, 다시 만났을 때 알아볼 수 있게 해주는 우리 동네 고양이 도감이다.

사용자 모집 문구:

> 사진첩에 묻혀 있던 우리 동네 고양이들, 이제 냥도감에 하나씩 모아보세요.

경쟁 서비스와의 차이는 다음처럼 정리한다.

| 비교 대상 | 한계 | 냥도감의 차이 |
| --- | --- | --- |
| 사진첩 | 사진은 남지만 고양이별 이력과 특징을 찾기 어렵다 | 사진을 고양이 개체별 기록으로 바꾼다 |
| 인스타그램 | 순간 공유에는 좋지만 지속 관찰 기록에는 맞지 않는다 | 좋아요보다 기록을, 노출보다 안전을 우선한다 |
| 동네 단톡방 | 대화가 지나가면 정보가 흘러간다 | 고양이별로 기록을 모아 나중에도 찾을 수 있다 |
| 커뮤니티 게시판 | 글 중심이라 개체별 이력이 누적되기 어렵다 | 한 마리마다 별도 기록 페이지와 재관찰 타임라인을 만든다 |

MVP에서도 반드시 보여야 하는 차이는 다음 6가지다.

| 반드시 보일 차이 | 이유 |
| --- | --- |
| 고양이별 개별 페이지 | 사진 앱이 아니라 도감처럼 느껴진다 |
| 다시 만난 기록 | 반복 사용 이유가 생긴다 |
| 별명/특징 기록 | 사용자가 고양이를 구분하게 된다 |
| 대략 위치만 표시 | 안전한 서비스라는 신뢰를 만든다 |
| 내 도감 보기 | 수집과 완성의 재미를 만든다 |
| 같은 고양이 후보 | "이 앱이 나를 도와준다"는 느낌을 만든다 |

현재 구현은 고양이별 페이지, 내 도감, 공유 도감, 대략 위치 표시, 재발견 기록 기반을 갖췄으므로 방향은 적합하다. 다만 사용자가 냥도감을 선택해야 하는 가장 강한 이유인 **재관찰의 깊이와 같은 고양이 후보 추천**은 아직 약하다. 따라서 다음 개발 우선순위는 꾸미기/구독 확장보다 `재관찰 입력`, `동네·털색·검색 기반 후보 추천`, `변화 메모와 관찰 태그`, `안전 위치 문구 정합성`을 먼저 강화한다.

포지셔닝 문장:

> 인스타그램은 고양이를 보여주는 곳이고, 사진첩은 고양이를 보관하는 곳이다. 냥도감은 고양이를 기억하는 곳이다.

## 3. 핵심 가치

1. 길고양이 만남을 도감처럼 쌓는다.
   - 고양이를 `No.001` 같은 카드 단위로 수집한다.
   - 털색, 성격 태그, 발견 지역, 메모, 사진을 기록한다.
   - 같은 고양이를 다시 만나면 새 고양이로 만들지 않고 재발견으로 남긴다.

2. 내 기록과 동네 공유 기록을 분리한다.
   - 공유 도감 전체는 동네 사람들이 함께 채운다.
   - 내 도감은 내가 실제로 발견한 고양이만 모아 보여준다.
   - 홈과 지도는 "공유 전체"와 "내 수집"을 명확히 나눠 표현한다.

3. 정확 위치는 내부 자산, 공개 위치는 흐림 정보로 분리한다.
   - 사용자가 위치 수집에 동의한 관찰 기록은 정확한 GPS/EXIF 좌표와 정확도, 수집 출처를 내부 민감 데이터로 저장한다.
   - 공개 도감, 일반 지도, 소셜 화면에는 정확 좌표를 직접 노출하지 않고 격자, 행정동, 지역명, 반경 기반 힌트만 보여준다.
   - 구조/TNR/활동가 검증처럼 정확 위치가 필요한 경우에는 인증된 권한, 목적 제한, 접근 로그를 전제로 제한 제공한다.
   - 사진 파일은 저장 전 EXIF를 제거하거나 재인코딩하지만, 동의받은 위치값은 사진 메타데이터가 아니라 별도 관찰 위치 데이터로 보관한다.

4. 수익화는 수집 제한이 아니라 꾸미기와 공유 확장에 둔다.
   - 무료 사용자는 촬영, 등록, 기본 도감 조회, 내 발견 지도까지 사용할 수 있다.
   - `냥꾸러미` 사용자는 공유 지도, 공개 도감 탐색, 좋아요/팔로우, 프리미엄 꾸미기를 사용한다.

## 4. 주요 사용자

### 동네 산책 기록형 사용자

- 산책 중 만난 고양이를 사진과 이름으로 기억하고 싶다.
- 정확한 위치를 공개하는 것은 부담스럽다.
- 내가 몇 마리를 봤는지, 다시 만난 고양이가 누구인지 보고 싶다.

### 수집/꾸미기 선호 사용자

- 도감 표지, 대표 고양이, 배지, 시즌 도장으로 내 컬렉션을 꾸미고 싶다.
- 공개 도감으로 다른 사람에게 내 수집 결과를 보여주고 싶다.
- 시즌 이벤트와 한정 장식에 반응할 가능성이 높다.

### 동네 탐색형 사용자

- 다른 사람들이 어느 동네에서 어떤 고양이를 봤는지 궁금하다.
- 다른 사용자의 공개 도감을 둘러보고 좋아요/팔로우를 하고 싶다.
- 공유 지도와 동네 도감 랭킹이 주요 사용 동기가 된다.

### 케어테이커/활동가 사용자

- 돌보는 고양이의 반복 출현, TNR 여부, 건강 이상 의심 기록을 체계적으로 관리하고 싶다.
- 정확 위치는 공개하지 않되, 본인 그룹이나 인증 활동가 범위에서는 현장 조치에 필요한 위치 정보를 확인해야 한다.
- 중복 개체 병합, 건강 이상 제보 검증, TNR 후보 정리 같은 운영 기능이 필요하다.

### 지자체/단체/연구 사용자

- 격자/행정동 단위의 반복 관찰 개체 수, 미중성화 추정, 자묘 출현, 건강 이상 제보를 집계하고 싶다.
- 개별 고양이의 실시간 위치보다 TNR 우선순위, 구조/치료 큐, 월간 리포트 같은 의사결정 자료가 필요하다.
- 정확 위치 접근은 계약, 권한, 감사 로그, 목적 제한이 있는 별도 대시보드에서만 허용한다.

## 5. 현재 구현된 제품 범위

### 인증

- Google/Kakao OAuth 로그인 흐름을 제공한다.
- Supabase Auth 세션을 기반으로 앱 리소스를 불러온다.
- 프로필 수정, 로그아웃, 세션 복원 흐름이 있다.
- 앱 시작 시 스플래시 화면을 보여주고, 세션 복원 중에는 로그인 상태 확인 UI를 노출한다.
- OAuth redirect URI는 Expo Go와 development build를 구분해 `EXPO_PUBLIC_OAUTH_REDIRECT_URI`와 `catdex://auth/callback` 기준으로 처리한다.

### 프로필/계정

- 사용자는 닉네임을 2자 이상 20자 이하로 수정할 수 있다.
- 프로필 이미지는 갤러리에서 정사각형으로 선택해 `profile-images` Storage 버킷에 업로드한다.
- Supabase Auth metadata와 `profiles` 테이블을 함께 갱신해 앱 내 표시명과 공개 도감 표시명을 맞춘다.

### 홈

- 오늘 발견, 내 도감 수집, 공유 도감 전체, 내 최고 재발견 요약을 보여준다.
- 최근 발견 고양이 카드와 `고양이 만남 기록하기` CTA를 제공한다.
- 공개 화면에는 동네 단위로만 보이고, 내부 관찰 데이터는 권한에 따라 보호된다는 원칙을 화면에 노출한다.

### 촬영/등록

- 카메라 촬영과 갤러리 사진 선택을 지원한다.
- 현재 구현은 갤러리 사진의 EXIF GPS로 가까운 동네를 제안하고, 정확 좌표는 저장하지 않는다.
- 목표 전환 후에는 사용자의 위치 수집 동의가 있을 때 촬영 GPS 또는 EXIF GPS를 내부 관찰 위치로 저장하고, 공개용 지역/격자를 별도로 계산한다.
- 사진 파일은 저장 전 재인코딩해 EXIF 위치정보가 파일에 남지 않도록 처리한다.
- 새 공유 고양이 등록, 기존 고양이 재발견, 미확인 제보 등록 흐름이 있다.
- 기존 고양이 재발견은 단순 카운트 증가가 아니라 오늘 다시 만난 동네, 변화 메모, 빠른 관찰 태그, 사진을 함께 저장하는 방향으로 보강한다.
- 같은 고양이 후보는 MVP에서 동네, 털색, 이름/특징 검색을 먼저 사용하고, 이후 사진 유사도와 활동가 검증으로 확장한다.

### 도감

- 내가 수집한 고양이 목록을 카드 그리드로 보여준다.
- 검색, 털색 필터, 도감 진행률을 제공한다.
- 미확인 목격 제보는 잠금/단서 카드로 노출한다.
- 도감 표지는 `고양이 서랍` 설정을 반영한다.

### 상세/재발견/신고

- 고양이 상세에서 발견 타임라인을 확인한다.
- 기존 고양이에 대해 `다시 만난 기록`을 추가할 수 있다.
- 재발견 기록에는 동네 단위 장소와 변화 메모를 남겨 사진첩이 아니라 기억장처럼 쌓이게 한다.
- 잘못된 정보, 위치 위험, 중복 등 신고 기반이 마련되어 있다.

### 지도

- 무료 사용자는 `내 발견 지도`에서 내가 발견한 고양이만 지역 단위로 본다.
- `냥꾸러미` 사용자는 `공유 도감 지도`에서 동네 단위 공유 고양이를 볼 수 있다.
- Kakao Map WebView를 사용하되 실패 시 지역 단위 fallback UI를 제공한다.
- 장기적으로 일반 지도는 흐림 처리된 공개 위치만 사용하고, 정확 위치 지도는 인증 활동가/단체/운영자 대시보드의 제한 기능으로 분리한다.

### MY

- 프로필, 레벨, 수집 수, 재발견 횟수, 배지, 대표 고양이를 보여준다.
- 프로필 수정, 고양이 서랍, 동네 도감 랭킹, 탐험 기록, 내가 공유한 도감, 좋아요한 도감, 알림 설정 진입점을 제공한다.
- 탐험 기록은 내 수집 고양이의 최근 만남과 재발견 횟수를 보여준다.
- 내가 공유한 도감은 공개 상태, 대표 고양이, 배지 수를 확인하고 고양이 서랍으로 이어진다.
- 좋아요한 도감은 랭킹/공개 도감 탐색에서 저장한 공개 도감 목록으로 돌아가는 진입점이다.

### 고양이 서랍

- 도감 표지, 표지 문구, 공개 설정, 우리 도감 주인공, 골목 배지, 냥발 도장을 관리한다.
- 무료 사용자는 기본 표지와 대표 고양이 1마리, 제한된 배지/도장 진열을 사용한다.
- `냥꾸러미` 사용자는 프리미엄 표지, 대표 고양이 최대 3마리, 시즌 냥발 도장을 사용할 수 있다.
- 현재 표지 배경 자산은 `노을 언덕`, `여름 강가`, `이끼숲 산책`, `비 오는 골목정원`, `단풍 신사길`, `눈 내린 마을` 계열로 구성되어 있다.
- 시즌 도장 자산은 봄 산책, 여름 그늘, 가을 골목, 겨울 담요 계열로 앱 번들에 포함되어 있다.

### 공개 도감/소셜

- 사용자는 자신의 도감을 공개하거나 비공개로 설정할 수 있다.
- 공개 도감 상세, 동네 도감 랭킹, 좋아요, 팔로우 기능의 Supabase RPC와 클라이언트 화면이 있다.
- 무료 사용자는 공개 도감 탐색 전에 `냥꾸러미` 안내 화면을 본다.

### 알림

- 알림 설정 화면, 로컬 탐험 리마인더, Expo Push Token 등록 기반이 있다.
- Supabase에는 알림 설정, 기기 토큰, 알림 이벤트 큐 테이블이 있다.
- Edge Function 기반 `notification-dispatch`가 서버 푸시 발송기로 준비되어 있다.

### 결제 준비

- RevenueCat SDK 연동 코드가 있다.
- Entitlement ID는 `nyangkkureomi`를 기준으로 한다.
- 클라이언트에는 RevenueCat public SDK key만 노출하고, secret과 Supabase `service_role` key는 Edge Function/서버 환경으로 분리한다.
- 현재 RevenueCat 키와 스토어 상품이 없으면 결제 준비 상태 안내를 보여준다.

### 앱 빌드/운영 설정

- Expo 앱 이름은 `냥도감`, scheme은 `catdex`다.
- iOS bundle identifier와 Android package는 `com.persimmontree.catdex`다.
- EAS development build는 `developmentClient: true`, `distribution: internal` 기준으로 준비되어 있다.
- Android 권한은 카메라, 오디오, 알림 권한을 포함한다.
- Kakao Map JavaScript key, Kakao native app key, Supabase URL/publishable key, RevenueCat public SDK key가 환경 설정의 핵심 값이다.

## 6. 핵심 화면 기획

### 6.1 홈

목표: 사용자가 앱을 켰을 때 오늘의 탐험 상태와 다음 행동을 즉시 알게 한다.

주요 콘텐츠:

- 공유 도감 전체 수
- 내 도감 수집 수
- 오늘 공유 도감에 새로 발견된 고양이 수
- 내 최고 재발견 고양이
- 최근 발견 고양이 3마리
- 촬영/등록 CTA

개선 방향:

- 우측 상단 알림 아이콘을 알림 설정 또는 알림함으로 연결한다.
- `최근 발견한 고양이 > 더보기`를 내 탐험 기록 또는 도감 필터로 연결한다.

### 6.2 촬영

목표: 새 고양이 등록, 기존 고양이 재발견, 미확인 제보를 한 흐름에서 처리한다.

주요 액션:

- 카메라 촬영
- 갤러리 사진 선택
- 기존 고양이 재발견
- 새 고양이 등록
- 미확인 제보 등록

정책:

- 이름과 지역이 있으면 새 고양이 등록 가능
- 지역만 있으면 미확인 제보 가능
- 사용자가 위치 수집에 동의하면 촬영 GPS 또는 사진 EXIF 좌표를 내부 관찰 위치로 저장
- 공개용 지역/격자는 시스템이 자동 계산하고 사용자가 최종 확인/수정 가능
- 사진 파일의 EXIF 좌표는 저장하지 않고, 필요한 위치값은 별도 DB 필드로만 보관
- 위치 수집에 동의하지 않으면 현재처럼 지역명 기반 수동 기록으로 대체

개선 방향:

- 기존 고양이 후보를 단순 최근 3마리에서 털색/지역/최근 발견 기반 추천으로 고도화한다.
- 중복 고양이 가능성을 안내하는 후보 매칭 UX를 추가한다.
- 귀끝 표식, 자묘/성묘, 건강 이상 의심, 주변 환경 태그를 빠른 입력으로 추가해 TNR/생태 데이터 품질을 높인다.

### 6.3 도감

목표: 내가 수집한 고양이를 생물 도감처럼 정리해 보여준다.

주요 콘텐츠:

- 도감 표지
- 진행률
- 고양이 카드
- 미확인 제보 단서 카드
- 검색/필터
- 고양이 서랍 진입
- 동네 도감 랭킹 진입

개선 방향:

- 미확인 제보 카드를 클릭했을 때 "찾으러 가기" 맥락의 촬영 진입을 제공한다.
- 필터를 털색 외에 희귀도, 최근 발견, 재발견 많은 순으로 확장한다.

### 6.4 고양이 상세

목표: 한 고양이의 정체성과 만남의 역사를 보여준다.

주요 콘텐츠:

- 대표 사진
- 이름, 번호, 털색, 희귀도
- 관계 레벨
- 발견/재발견 횟수
- 발견 타임라인
- 신고
- 재발견 기록 추가

개선 방향:

- 대표 사진 변경/사진 추가 기능을 업로더 권한과 함께 설계한다.
- 중복 신고가 누적된 고양이는 관리자 병합 후보로 표시한다.

### 6.5 지도

목표: 일반 사용자는 흐림 처리된 동네/격자 단위로 탐색하고, 인증된 운영 주체는 목적 제한 하에 정확 위치를 활용하게 한다.

무료 사용자:

- 내 발견 지도만 조회
- 내가 발견한 지역과 고양이 이름 목록 표시
- 공유지도 보기 버튼 클릭 시 `냥꾸러미` 안내

`냥꾸러미` 사용자:

- 공유 도감 지도 조회
- 동네별 공유 고양이 수와 이름 목록 표시
- 정확한 좌표가 아니라 지역 중심/반경, 격자, 행정동 수준으로 표시

인증 활동가/단체/운영자:

- TNR 후보, 건강 이상 제보, 콜로니 관리처럼 현장 조치가 필요한 건에 한해 정확 위치 접근
- 접근 사유 기록, 사용자 권한 확인, 접근 로그 저장
- 민감 위치는 기본 비공개이며, 필요 시 일회성/만료형 공유로 제한

개선 방향:

- Kakao Map WebView 실패 원인과 도메인 설정을 점검한다.
- 지도 fallback UI를 단순 리스트가 아니라 지역별 카드/미니맵 형태로 강화한다.
- 일반 앱 지도와 운영자/활동가용 정확 위치 지도를 분리한다.

### 6.6 MY

목표: 내 활동, 내 도감 상태, 설정 진입점을 모은다.

주요 콘텐츠:

- 프로필 이미지/닉네임
- 레벨/진행률
- 도감 수집, 재발견, 공유, 연속 탐험 통계
- 획득 배지
- 우리 도감 주인공
- 고양이 서랍
- 알림 설정
- 공개/좋아요/탐험 기록 메뉴

개선 방향:

- `공유 수`는 현재 대표 고양이 수처럼 보일 수 있으므로 공개 도감 수치 또는 공유 공개 상태와 맞게 용어를 정리한다.
- `연속 탐험`은 실제 streak 계산이 붙기 전까지 `획득 도장` 등 현재 데이터에 맞는 명칭으로 바꾼다.

### 6.7 로그인/스플래시

목표: 사용자가 앱의 감성을 먼저 이해하고, 계정 기반 기록 저장을 자연스럽게 시작하게 한다.

주요 콘텐츠:

- 앱 스플래시
- 세션 복원 상태
- 카카오 로그인
- 구글 로그인
- OAuth 실패/취소 오류 메시지

개선 방향:

- 로그인 전에도 기능 프리뷰를 보여줄지 결정한다.
- OAuth redirect 설정 오류를 사용자가 이해할 수 있는 문구로 분리한다.
- 약관, 개인정보 처리방침, 위치 안전 원칙 링크를 로그인 화면 또는 설정 화면에 추가한다.

### 6.8 프로필 수정

목표: 공개 도감과 MY에 표시되는 사용자 정체성을 관리한다.

주요 콘텐츠:

- 현재 프로필 이미지
- 갤러리 이미지 선택
- 닉네임 입력과 길이 검증
- 저장 중/오류 상태

정책:

- 닉네임은 2자 이상 20자 이하로 제한한다.
- 프로필 이미지는 `profile-images` 버킷에 저장하고 공개 URL을 사용한다.
- 공개 도감 방문 화면에서는 이메일이나 OAuth provider 원본 값은 노출하지 않는다.

### 6.9 생태 데이터/활동가 대시보드

목표: B2C 도감 앱에서 쌓인 관찰 기록을 TNR, 건강 이상 제보, 콜로니 관리, 지자체 리포트로 확장한다.

주요 콘텐츠:

- TNR 후보 큐: 귀끝 표식 없음/모름, 반복 출현, 자묘 동반, 신규 유입 기준
- 건강 이상 제보 큐: 상처 의심, 절뚝임, 눈 이상, 너무 마름, 임신/수유 의심, 새끼 단독 발견
- 콜로니 관리: 급식소, 반복 출현 개체, 최근 미출현, 공사/재개발 위험 플래그
- 지역 현황판: 반복 식별 개체 수, TNR 추정률, 자묘 출현률, 신규 유입률
- 월간 리포트: 관찰 기록, 우선 조치 후보, 데이터 신뢰도, 민감 위치 제외 요약

정책:

- 일반 사용자에게는 정확 위치를 노출하지 않는다.
- 인증 활동가/단체/지자체 권한은 별도 승인과 목적 제한을 전제로 한다.
- 모든 정확 위치 접근은 감사 로그로 남긴다.
- 건강 이상 기능은 질병 진단이 아니라 `전문가 확인이 필요한 의심 제보`로 표현한다.

### 6.10 데이터 활용 화면

초기 비전 문서의 데이터 활용 아이디어는 별도 앱을 새로 만드는 것이 아니라, 현재 냥도감에서 쌓이는 관찰 기록을 단계적으로 확장해 만든다.

#### TNR 우선순위 지도

| 지표 | 계산 방향 | 활용 |
| --- | --- | --- |
| 미중성화 추정 개체 수 | 귀끝 표식 없음/모름 중 반복 관찰 개체 수 | TNR 후보 발굴 |
| TNR 추정률 | 귀끝 표식 있음 개체 / 전체 반복 식별 개체 | 구역별 관리 수준 비교 |
| 자묘 출현률 | 자묘 태그 관찰 수 / 전체 관찰 수 | 번식 활성 구역 탐지 |
| 신규 개체 유입률 | 최근 N일 내 처음 등장한 개체 비율 | 유기·이동·재개발 영향 탐지 |
| 반복 출현 안정도 | 같은 고양이의 동일 구역 반복 관찰 비율 | 콜로니/서식권 추정 |

지도 색상은 단순히 “고양이가 많다”가 아니라 “관리 우선순위”를 표현한다. 일반 사용자 지도는 격자/행정동 단위로만 보여주고, 정확 위치는 인증 권한이 있는 운영 화면에서만 확인한다.

#### 사진 기반 개체수 추정

- 같은 고양이 후보 추천은 이미지 유사도, 위치·시간 필터, 사용자 확인, 활동가 검증 순서로 고도화한다.
- 앱은 절대 “같은 고양이 확정”이라고 말하지 않고 “전에 만난 고양이와 비슷해요” 같은 후보형 문구를 사용한다.
- 잘못 병합된 고양이를 되돌릴 수 있도록 병합/분리 이력을 저장한다.

#### 구조·치료 우선순위 큐

- 건강 이상 태그는 상처 의심, 절뚝임, 눈 이상, 너무 마름, 임신/수유 의심, 새끼 단독 발견으로 시작한다.
- 앱은 질병명이나 응급 판단을 단정하지 않고, 반복 관찰과 사진 품질을 기반으로 `전문가 확인 필요` 큐에 쌓는다.
- 구조 요청 위치는 일반 공개하지 않고, 검증된 활동가/단체 권한에서만 접근한다.

#### 급식소·콜로니 관리

- 반복 출현 위치, 시간대, 주변 환경, 급식소 주변 여부, 공사/재개발 태그를 콜로니 추정에 사용한다.
- 케어테이커 그룹은 비공개 콜로니 노트, 최근 미출현 고양이, 신규 미중성화 추정 고양이 알림을 사용할 수 있다.
- 급식소 정보는 갈등 유발 가능성이 높으므로 기본 비공개 또는 그룹 공개로만 다룬다.

#### 민원·갈등 완화 리포트

- “고양이가 많다” 같은 주관적 주장 대신 중복 제거된 반복 식별 개체 수, 자묘 출현률, TNR 추정률, 신규 유입률을 제공한다.
- 개별 고양이 좌표가 아니라 집계 통계와 격자/행정동 단위로만 제공한다.
- 특정 돌봄자, 급식소, 고양이를 공격하는 도구가 되지 않도록 커뮤니티 정책과 신고 기능을 함께 둔다.

#### 도시생태·연구 데이터

- 연구용 export는 익명화·집계 데이터만 기본 제공한다.
- 연구 질문은 서식환경별 출현, 계절별 자묘 출현, TNR 집중 구역의 자묘 출현 변화, 재개발/공사 구역의 출현 패턴 변화부터 시작한다.
- 한국 도시 데이터로 해석할 수 없는 해외 연구 수치를 앱 안에서 직접 단정하지 않는다.

## 7. Supabase 데이터 모델 요약

### 계정/프로필

- `profiles`: 앱 표시용 닉네임, 이메일, provider, 프로필 이미지 URL
- `profile-images` Storage bucket: 공개 프로필 이미지 파일

주요 처리:

- 신규 Auth 사용자 생성 시 `private.handle_new_user` 트리거로 `profiles` row를 만든다.
- 프로필 수정 시 Supabase Auth metadata와 `profiles`를 함께 갱신한다.

### 공유 도감

- `cats`: 공유 고양이 엔티티
- `cat_encounters`: 사용자별 발견 기록
- `user_cat_collections`: 사용자별 수집 상태
- `regions`: 지역 기준 정보
- `cat_regions`: 공유 고양이와 지역 연결
- `cat_photos`: 고양이/발견 사진
- `cat_sightings`: 미확인 목격 제보
- `reports`: 신고

주요 RPC:

- `create_cat`: 공유 고양이 생성, 첫 발견 기록, 내 수집 상태, 지역 연결을 함께 생성
- `record_cat_encounter`: 기존 공유 고양이 재발견 기록과 내 수집 상태 갱신
- `create_cat_sighting`: 미확인 목격 제보 생성

### 위치/관찰 데이터 전환 목표

현재 DB는 `region_name`, `regions`, `cat_regions` 중심으로 동작한다. 초반 비전 문서를 기준으로 다음 단계에서는 정확 위치와 공개 위치를 분리한다.

- `private.sighting_locations`: 관찰별 정확 좌표, GPS 정확도, 수집 출처, 촬영/관찰 시각, 민감 플래그
- `public_location_grid`: 일반 화면과 공개 API에서 사용할 격자/행정동/지역 힌트
- `location_consents`: 사용자별 위치 수집 동의, 목적, 철회 시각
- `location_access_logs`: 정확 위치 조회자, 조회 대상, 목적, 시각, 요청 경로
- `sighting_tags`: 귀끝 표식, 자묘/성묘, 건강 이상 의심, 주변 환경, 급식소 주변 여부
- `verification_status`: 미검증, 커뮤니티 검증, 활동가 검증, 운영자 확인

설계 원칙:

- 정확 좌표는 `public` 노출 테이블에 직접 두지 않고, 가능하면 `private` 스키마 또는 제한 RPC 뒤에 둔다.
- 일반 앱 화면과 공개 도감 RPC는 정확 좌표 대신 공개용 격자/지역만 반환한다.
- PostGIS 도입 여부는 구현 단계에서 결정하되, MVP 전환은 `lat`, `lng`, `accuracy_meters` numeric 필드로도 시작할 수 있다.
- 위치 동의를 철회하면 이후 정확 위치 수집을 중단하고, 기존 위치의 삭제/익명화 정책을 제공한다.

### TNR/생태 데이터 확장 목표

- `tnr_events`: 포획일, 수술일, 방사일, 수행 단체, 상태, 방사 후 재출현 확인
- `colonies`: 반복 출현 구역, 관리 그룹, 추정 개체 수, TNR 추정률, 위험 플래그
- `health_reports`: 건강 이상 의심 제보, 사진, 긴급도, 검증 상태
- `cat_identity_candidates`: 같은 고양이 후보, 이미지 유사도, 위치/시간 필터, 사용자 확인 결과
- `cat_merge_events`: 개체 병합/분리 이력과 검증자

이 확장 데이터는 일반 B2C 도감 UX를 해치지 않도록 빠른 태그 입력과 후보 확인 흐름으로만 수집하고, 활동가/단체/운영자 화면에서 정제한다.

### 데이터 품질 관리 목표

- `observation_confidence`: 관찰 기록의 신뢰도. 사진 선명도, 위치 정확도, 태그 완성도, 반복 관찰 여부를 반영한다.
- `identity_confidence`: 같은 고양이 후보의 신뢰도. 이미지 유사도, 시간차, 거리, 사용자 확인, 활동가 검증을 반영한다.
- `tag_confidence`: 귀끝 표식, 자묘 여부, 건강 이상 의심 같은 태그의 확실함/모름/추정을 분리한다.
- `duplicate_photo_hashes`: 같은 사진 재업로드 또는 도용 업로드를 감지하기 위한 perceptual hash 저장소.
- `verification_reviews`: 활동가/운영자의 검증 결과와 이의제기 이력.

데이터 분석 화면은 단순 관찰 수만 보여주지 않는다. 사용자 수, 관찰 수, 반복 식별 개체 수, 데이터 부족 구역, 야간/주간·평일/주말 편향을 함께 보여준다.

## 8. 권한/RLS 원칙

- `public` 스키마 테이블은 RLS를 활성화한다.
- 공유 고양이는 인증 사용자에게 조회를 허용하되, 개인 발견 기록은 공개 가능 범위와 본인 기록을 분리한다.
- 사용자는 자신의 수집 상태, 대표 고양이, 도감 프로필, 알림 설정, 기기 토큰만 수정할 수 있다.
- 공개 도감 RPC는 이메일, provider, 정확 좌표, 비공개 발견 기록을 반환하지 않는다.
- 정확 위치 데이터는 일반 `select *` 경로로 조회되지 않게 private schema, 제한 RPC, 역할 기반 정책, 감사 로그를 함께 적용한다.
- 인증 활동가/단체/지자체 권한은 `user_metadata`가 아니라 서버 검증 가능한 role/organization membership 테이블 또는 app metadata 기반으로 관리한다.
- 정확 위치 조회 RPC는 조회 목적, 대상, 요청자를 `location_access_logs`에 남긴 뒤 결과를 반환한다.
- 민감 위치, 새끼 은신처, 건강 이상/구조 제보, 학대 위험 신고는 기본 비공개이며, 일반 공개 지도/소셜 화면에 포함하지 않는다.
- 결제 검증 secret, RevenueCat secret, Supabase `service_role` key는 클라이언트에 두지 않는다.
- `user_metadata`는 사용자가 바꿀 수 있으므로 RLS 권한 판단에 사용하지 않는다.
- `security definer` 함수는 가능한 `private` 스키마에 두고, 공개 RPC는 필요한 입력과 반환값만 노출한다.
- UPDATE 가능한 테이블은 SELECT 정책이 함께 있어야 하며, Storage upsert가 필요해질 경우 INSERT/SELECT/UPDATE 정책을 함께 검토한다.

### 8.1 안전·프라이버시 정책

정확 위치를 내부 저장하는 방향으로 전환하더라도, 공개 정책은 보수적으로 유지한다.

| 데이터 | 저장 | 일반 공개 | 제한 공개 |
| --- | --- | --- | --- |
| 정확 GPS/EXIF 위치 | 동의 기반 내부 저장 | 금지 | 인증 활동가/단체/운영자만 목적 제한 조회 |
| 공개 지도 위치 | 격자/행정동/지역명 | 허용 | 동일 |
| 구조/건강 이상 위치 | 제한 저장 | 금지 | 검증된 단체만 조회 |
| 새끼 은신처/학대 위험 위치 | 민감 위치로 저장 | 금지 | 운영자 승인 후 제한 조회 |
| 실시간 발견 시각 | 저장 가능 | 지연 또는 일 단위 | 필요 시 제한 조회 |

사진 안전 처리는 다음을 목표로 한다.

- 업로드 전 EXIF 제거 또는 재인코딩
- 사람 얼굴/차량 번호판 자동 블러 도입
- 집 현관, 상호, 주소가 뚜렷한 사진은 공개 제한
- 원본 사진과 공개용 이미지 접근 권한 분리
- 민감 신고 시 즉시 비공개 전환

### 8.2 악용 방지와 금지 방향

| 위험 기능 | 대체 설계 |
| --- | --- |
| 실시간 고양이 위치 지도 | 시간 지연 + 격자/행정동 히트맵 |
| 희귀 고양이 위치 랭킹 | 개인 도감 중심 보상 |
| 특정 장소로 가서 촬영하는 퀘스트 | 이미 만난 고양이 기록 품질 보상 |
| `가서 잡기` 또는 포획 유도 버튼 | 인증 단체 구조/TNR 요청만 |
| 정확 좌표 공유 링크 | 권한 기반 일회성/만료형 공유 |
| 인기 스팟 공개 | 지역 단위 통계만 표시 |

건강 이상 기능은 의료 진단처럼 보여서는 안 된다. 앱 문구는 `상처 의심`, `눈 이상 의심`, `전문가 확인 필요` 수준으로 제한하고, 병명 추정이나 약물 추천을 금지한다.

### 8.3 커뮤니티 운영 정책

금지:

- 고양이 정확 위치 댓글 공유
- 무단 포획, 이주방사, 사유지 침입 유도
- 학대·혐오 표현
- 특정 급식소, 돌봄자, 주민 공격
- 치료법 단정 또는 약물 추천

허용:

- 대략적인 관찰 기록 공유
- 귀끝/TNR 여부에 대한 후보형 제보
- 건강 이상 의심 제보
- 공식 구조/TNR 신청 안내
- 공존을 위한 위생관리 정보 공유

## 10. 주요 사용자 플로우

### 첫 고양이 등록

1. 로그인
2. 홈에서 `고양이 만남 기록하기`
3. 촬영 또는 갤러리 선택
4. 위치 수집 동의 상태를 확인하고, 동의 시 GPS/EXIF 좌표를 내부 관찰 위치로 준비
5. 공개용 지역/격자를 자동 계산하고 사용자가 동네 표시를 확인
6. 이름, 털색, 성격 태그, 귀끝/자묘/건강/환경 태그, 메모 입력
7. `create_cat` RPC 호출
8. 공유 도감, 내 도감, 내부 관찰 위치, 공개용 지역/격자를 함께 반영
9. 도감 화면에서 카드 확인

### 기존 고양이 재발견

1. 촬영 후 기록 설정 진입
2. 동네, 털색, 이름/특징 검색으로 기존 고양이 후보를 좁히거나 상세 화면에서 재발견 액션 선택
3. 동의된 정확 위치와 공개용 지역/격자를 함께 준비
4. 오늘 다시 만난 동네, 변화 메모, 빠른 관찰 태그, 사진을 `record_cat_encounter` RPC로 저장
5. 고양이 전체 발견 수, 내 재발견 수, 반복 출현 위치 데이터 갱신
6. 홈/MY 통계와 향후 TNR/콜로니 분석에 반영

### 미확인 제보

1. 고양이 이름을 확정하기 어렵지만 관찰 기록을 남기고 싶음
2. 촬영/갤러리 후 동의된 정확 위치, 공개용 지역/격자, 메모, 관찰 태그 입력
3. `create_cat_sighting` RPC 호출
4. 일반 도감에는 미확인 단서 카드로, 운영 화면에는 검증 대기 제보로 노출
5. 이후 실제 고양이 등록, 같은 고양이 후보 연결, 활동가/운영자 확인 흐름으로 연결

### TNR/건강 이상 제보

1. 사용자가 귀끝 표식 없음/모름, 자묘, 건강 이상 의심 같은 태그를 선택
2. 앱은 진단이나 포획을 유도하지 않고 `전문가 확인 필요` 문구를 보여줌
3. 반복 출현, 위치 안정도, 사진 선명도, 태그 신뢰도에 따라 후보 큐에 쌓임
4. 인증 활동가/단체가 정확 위치 접근 사유를 남기고 제보를 검증
5. TNR 예정, 구조/치료 확인, 보류, 오탐 같은 상태로 처리

### 고양이 서랍 꾸미기

1. MY 또는 도감 표지에서 `고양이 서랍` 진입
2. 표지 문구, 공개 여부, 테마, 대표 고양이, 배지/도장 설정
3. 무료 제한을 넘는 아이템 선택 시 `냥꾸러미` 안내
4. 저장 후 도감 표지와 MY 대표 영역에 반영

### 공개 도감 탐색

1. 도감 또는 MY에서 동네 도감 랭킹 진입
2. 무료 사용자는 `냥꾸러미` 안내
3. 구독 사용자는 공개 도감 랭킹 조회
4. 공개 도감 상세 진입
5. 좋아요/팔로우 토글
6. 소셜 알림 이벤트 생성

### 빠른 고양이 기록 PRD

목표: 사용자가 길고양이를 발견한 순간 10초 이내에 사진과 기본 관찰값을 남길 수 있게 한다.

입력:

- 사진 1장 이상
- 위치 동의 기반 자동 좌표 또는 수동 지역
- 발견 시각 자동 기록
- 귀끝 표식, 자묘/성묘, 건강 이상 의심, 주변 환경 태그
- 이름/메모는 선택 또는 후속 보완 가능

출력:

- 새 관찰 기록 생성
- 기존 고양이 후보 추천
- 개인 도감 카드 또는 미확인 제보 업데이트
- 공개용 격자/행정동 계산

예외:

- 위치 동의가 없으면 지역명 직접 입력으로 대체
- GPS 정확도가 낮으면 `대략 위치만 기록됨` 표시
- 사진에 사람/번호판/주소가 선명하면 공개 제한 또는 블러 처리
- 민감 위치로 판단되면 공개 범위 자동 제한

### 같은 고양이 후보 추천 PRD

목표: 도감 UX를 강화하고 중복 개체를 줄인다.

로직:

1. 이미지 유사도 또는 향후 임베딩 검색으로 후보를 찾는다.
2. 후보의 최근 관찰 위치와 시간 차이를 비교한다.
3. 사용자가 `같은 고양이 같아요`, `아니에요`, `모르겠어요` 중 하나를 선택한다.
4. 일정 신뢰도 이상이면 병합 후보로 등록한다.
5. 활동가/운영자 검증 후 병합하거나 분리 이력을 남긴다.

문구:

- 좋은 문구: `전에 만난 고양이와 비슷해요.`
- 피해야 할 문구: `이 고양이는 100% 같은 고양이입니다.`

### 민감 위치 보호 PRD

목표: 정확 위치 저장이 동물 안전과 사용자 안전을 해치지 않게 한다.

규칙:

- 일반 공개 좌표는 격자/행정동/지역명으로 흐림 처리한다.
- 새끼 은신처, 건강 이상, 학대 위험 신고는 자동 비공개로 둔다.
- 민감 위치 접근 시 사용자, 목적, 대상, 시각을 감사 로그에 저장한다.
- 공유 링크가 필요하면 만료 시간과 권한을 가진 일회성 링크로 제한한다.

## 11. 검증 기준

### 기본 자동 검증

- 클라이언트 변경 후 `cd catdex-client && npm run typecheck`
- Supabase 변경 후 RLS, RPC, grant 범위 확인
- 네이티브 권한이나 결제 관련 변경 후 EAS development build 또는 실제 기기 검증
- OAuth 변경 후 Expo Go redirect와 `catdex://auth/callback` development build redirect 모두 확인
- 정확 위치 저장 기능 변경 후 공개 RPC/공개 화면이 정확 좌표를 반환하지 않는지 SQL/RPC 단위 검증
- 정확 위치 조회 RPC는 권한 없는 사용자가 실패하고, 권한 있는 조회는 감사 로그를 남기는지 검증

### 수동 QA 기준

- 스플래시/로그인: 세션 복원, Google/Kakao OAuth, 실패 메시지
- 홈: 요약 수치와 최근 고양이 표시
- 도감: 검색, 필터, 빈 상태, 도감 표지
- 촬영: 카메라/갤러리, 위치 동의, EXIF/GPS 기반 공개 지역 계산, 등록 폼 필수값
- 지도: 내 발견 지도, 공유 지도 게이트, 정확 좌표 비노출, fallback 지도
- MY: 프로필, 프로필 수정, 탐험 기록, 공유한 도감, 좋아요한 도감, 고양이 서랍, 알림 설정, 소셜 메뉴
- 냥꾸러미: 무료 잠금, 구독 권한 사용자 접근, 결제 준비 상태
- 공개 도감: 랭킹, 상세, 좋아요/팔로우
- 알림: 권한 요청, 로컬 리마인더, 푸시 토큰 등록
- 활동가/운영자: 권한 승인, 정확 위치 조회, 접근 로그, TNR/건강 이상 후보 큐

## 12. 현재 리스크와 해결 과제

### 지도 안정성

Android 검증에서 Kakao Map WebView가 로드되지 않고 fallback 지역 지도가 표시된 기록이 있다. Kakao JavaScript 키, Web platform domain, Android WebView 네트워크 접근을 점검해야 한다.

### 정확 위치 저장 전환 리스크

현재 구현과 저장소 기준은 지역 단위 기록이다. 정확 GPS를 내부 저장하는 방향으로 전환하려면 위치정보 수집 동의, 수집 목적, 보관 기간, 철회/삭제, 권한별 접근, 감사 로그, 민감 위치 비공개 정책을 먼저 확정해야 한다. 이 작업 없이 단순히 좌표 컬럼만 추가하면 제품 신뢰와 법적 리스크가 커진다.

### 위치 공개/악용 리스크

정확 위치가 공개되면 무단 포획, 학대, 사유지 침입, 주민 갈등으로 이어질 수 있다. 일반 화면은 격자/행정동/지역명만 사용하고, 실시간 위치·새끼 은신처·건강 이상 제보 위치는 기본 비공개로 둔다.

### 실제 결제 검증

RevenueCat 연동 코드는 있으나 현재 public SDK key와 스토어 상품 설정이 없으면 패키지를 불러올 수 없다. EAS development build와 실제 기기 sandbox 환경에서 구매/복원을 검증해야 한다.

### OAuth와 딥링크 설정

Expo Go, development build, Supabase Redirect URLs, Kakao/Google 콘솔 redirect 설정이 서로 어긋나면 로그인 후 앱 복귀가 실패한다. 환경별 redirect URI를 운영 체크리스트로 고정해야 한다.

### 프로필 이미지 운영

`profile-images`는 public bucket이므로 업로드 가능한 파일 크기, MIME type, 사용자별 폴더 정책을 유지해야 한다. 프로필 이미지 삭제/교체 시 이전 파일 정리 정책은 아직 없다.

### 관리자 운영 도구 부재

신고 테이블과 신고 생성 흐름은 있지만, 중복 고양이 병합, 부적절 사진 숨김, 위치 위험 제보 처리 같은 관리자 화면은 없다.

### 중복 고양이 문제

공유 도감 구조에서는 같은 고양이가 여러 번 등록될 수 있다. 촬영/등록 단계에서 유사 지역, 털색, 최근 발견 사진을 기반으로 중복 후보를 제안하는 기능이 필요하다.

### 알림 발송 운영

알림 이벤트 큐와 Edge Function은 준비되어 있지만, 주기적 dispatch 호출, secret 관리, 실패 재시도 정책, 실제 기기 수신 검증이 필요하다.

### 통계 명칭 정합성

MY 화면의 일부 통계 문구가 실제 데이터 의미와 다를 수 있다. `공유 수`, `연속 탐험` 같은 항목은 실제 계산 모델이 확정되면 명칭을 조정해야 한다.

### 정책 문서 부재

로그인, 사진 업로드, 공개 도감, 알림, 결제, 정확 위치 저장이 들어가는 앱이므로 개인정보 처리방침, 이용약관, 위치정보 수집·이용 동의, 위치정보 보관/삭제 정책, 동물 안전 원칙 문서가 필요하다.

## 15. 기획 원칙

- 고양이와 사용자의 안전을 위해 정확 좌표는 동의 기반 내부 데이터로만 저장하고, 일반 공개 화면에는 격자/행정동/지역명 수준으로만 노출한다.
- 정확 위치 접근은 인증된 역할, 목적 제한, 감사 로그, 민감 위치 비공개 정책을 전제로 한다.
- 무료 사용자의 핵심 수집 경험은 막지 않는다.
- 구독 기능은 꾸미기, 공유, 시즌성 보상 중심으로 설계한다.
- 공유 도감은 모두가 보는 데이터이고, 내 도감은 내가 실제로 만난 데이터라는 구분을 모든 화면에서 유지한다.
- Supabase 접근은 클라이언트 public key와 RLS/RPC를 기준으로 하고, secret 기반 작업은 Edge Function 또는 서버 측 검증으로 분리한다.
- 게임화는 발견 경쟁이 아니라 기록의 질, 꾸준함, 안전한 반복 관찰에 보상한다.
- 앱은 의료 진단, 포획 지시, 민원 대결 도구처럼 보이면 안 된다.
- 장기 사업화는 B2C 도감과 꾸미기로 사용자 기반을 만들고, 활동가/단체 파일럿을 거쳐 B2B/B2G 리포트와 대시보드로 확장한다.
