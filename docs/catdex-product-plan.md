# 냥도감 제품 계획

## 제품 방향

냥도감은 사용자가 길고양이를 촬영하고, 다시 만난 기록을 쌓으며, 생물 도감처럼 개인 수집 목록을 완성해 가는 모바일 앱이다. B2C 앱은 쉬운 촬영, 기록, 수집, 개인 지도 확인에 집중한다.

장기적으로 관찰 데이터는 TNR, 돌봄, 지자체, 연구 협업에 활용될 수 있다. 다만 일반 사용자 화면은 반복 사용이 쉬운 기록 앱으로 유지하고, 기관/운영자용 기능은 별도 권한 화면이나 대시보드로 분리한다.

## 위치 정책

- 정확한 GPS/EXIF 위치는 사용자가 명시적으로 동의한 촬영/관찰 기록에 한해 내부 민감 데이터로 저장할 수 있다.
- 일반 사용자 화면, 지도, 공유 화면, 공개 API에는 정확 좌표를 직접 노출하지 않는다.
- B2C 앱의 지도와 목록은 동, 구, 격자, 반경 흐림 등 지역 단위 값만 사용한다.
- 정확 위치 저장을 도입할 때는 별도 전환 문서, Supabase 마이그레이션, RLS, 제한 RPC, 역할 기반 권한, 접근 목적 기록, 감사 로그 설계를 먼저 확정한다.
- Storage에 공개되는 이미지는 위치 EXIF가 노출되지 않도록 제거하거나 공개용 파생본을 사용한다.

## 현재 앱 표면

하단 탭은 4개다.

- 홈: 개인 발견 지도
- 도감: 수집한 고양이 2열 카드 그리드
- 촬영: 신규 고양이 등록, 미확인 제보, 기존 고양이 재발견 기록
- MY: 프로필, 수집/재발견 통계, 최근 만난 고양이, 탐험 기록, 알림 설정, 로그아웃

홈은 기존 대시보드가 아니라 개인 발견 지도다. 지도는 Kakao Maps WebView를 사용하되, 지도 로드가 실패하면 지역 단위 fallback UI와 원인 진단을 보여준다.

## 도감 화면

- 화면 상단 표지 영역은 두지 않는다.
- 기본 화면은 바로 고양이 카드 2열 그리드로 시작한다.
- 검색창과 필터칩은 기본 숨김 상태다.
- 상단 필터 아이콘을 누를 때만 검색창과 필터칩을 펼친다.

## MY 화면

- MY는 기본 수집 경험과 계정/알림 설정에 집중한다.
- 통계는 도감 수집 수, 재발견 횟수, 내 고양이 수, 다음 레벨 진행률을 중심으로 보여준다.
- 최근 만난 고양이와 탐험 기록 진입을 제공한다.

## Supabase 모델

유지하는 핵심 도메인:

- `profiles`: 사용자 기본 프로필
- `cats`: 고양이 기본 정보
- `cat_encounters`: 사용자 관찰/재발견 기록
- `user_cat_collections`: 사용자별 수집 상태
- `regions`, `cat_regions`, `region_cats`: 지역 단위 표시
- `cat_photos`, `cat_sightings`: 사진과 미확인 제보
- `notification_settings`, `notification_devices`, `notification_events`: 기본 알림 인프라

제거된 도메인은 새 기능에서 다시 참조하지 않는다. 수익화, 장식, 공개 탐색, 반응, 획득 보상 기능을 재도입하려면 별도 전환 문서를 먼저 작성하고 현재 위치 정책을 다시 검토한다.

## 검증 기준

- 클라이언트 기본 검증은 `cd catdex-client && npm run typecheck`다.
- 홈 진입 후 개인 발견 지도가 보여야 한다.
- 하단 탭은 홈, 도감, 촬영, MY만 보여야 한다.
- 선택된 하단 탭에는 연두색 둥근 테두리가 보여야 한다.
- 도감은 표지 없이 2열 카드 그리드로 시작해야 한다.
- 필터 아이콘을 누를 때만 검색/필터 UI가 열려야 한다.
- MY에는 제거된 장식, 공개 탐색, 반응, 획득 보상 메뉴가 없어야 한다.
- Supabase 제거 검증은 `supabase/verify_remove_customization_social_badges.sql`을 사용한다.
